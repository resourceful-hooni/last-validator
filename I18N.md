# I18N — 다국어(현지화) 사양 · 「마지막 검증자」

> 본 경험은 **한국어(ko)·영어(en)·중국어 간체(zh)·일본어(ja)** 4개 언어로 서비스한다.
> 모든 사용자 노출 텍스트는 **키 기반**으로 관리하며 하드코딩 금지. 정본 카피는 한국어(STORYBOARD)이고, 전체 번역은 `TRANSLATIONS.md`에 있다.

## 1. 지원 로케일
| 코드 | 언어 | 비고 |
|---|---|---|
| `ko` | 한국어 | **정본(source of truth)** |
| `en` | English | |
| `zh` | 中文(简体) | 간체. 번체(zh-Hant)는 향후 확장 여지 |
| `ja` | 日本語 | |

- 기본(default) 로케일: `ko`. 폴백 체인: `요청 로케일 → ko`.
- 전부 LTR(좌→우). RTL 불필요.

## 2. 로케일 결정 & 지속
1. URL 쿼리/경로 우선: `?lang=en` 또는 `/en/`(라우팅 택1, TECH_SPEC 참조).
2. 없으면 `localStorage('locale')`.
3. 없으면 `navigator.language`로 추정(`ko*→ko, zh*→zh, ja*→ja, 그 외→en` 또는 `en` 기본은 정책 선택; 권장: 매칭 없으면 `en`).
4. 사용자가 전환하면 `localStorage`에 저장 + URL 동기화. **진행 중 전환 허용**(현재 씬 유지, 텍스트만 교체).

## 3. 키 시스템
- 네임스페이스: `ui.*`(공통 크롬), `s0.*`~`s12.*`(장면), 분기 `s9.tragedy.*` / `s9.nearmiss.*` / `s9.averted.*`, 평결 `s10.verdict.*`.
- 파일: `src/locales/ko.json`, `en.json`, `zh.json`, `ja.json` — **동일 키 집합**(키 누락 시 빌드 경고).
- 보간(interpolation): `{n}`(진행), `{eff}` 기반 수치는 코드에서 계산 후 `{value}` 주입. 예) `s6.scoreEfficiency = "운영 효율 +{value}%"`.
- 강조 마크업: 본문은 토큰 `<em>`(강조), `<gn>`(초록), `<rd>`(빨강)만 허용. **번역에서도 동일 토큰 유지**(위치는 언어에 맞게 이동 가능).

## 4. 언어별 폰트 전략 (필수)
CJK는 자형이 달라 **언어별 폰트**가 필요하다. 로케일 전환 시 `<html lang>`와 본문 폰트를 함께 교체.
| 로케일 | 권장 폰트 스택 (전부 OFL/시스템) |
|---|---|
| `ko` | `'Pretendard','Apple SD Gothic Neo','Noto Sans KR',sans-serif` |
| `en` | `'Inter','Helvetica Neue',Arial,sans-serif` (또는 시스템 sans) |
| `zh` | `'Noto Sans SC','PingFang SC','Microsoft YaHei',sans-serif` |
| `ja` | `'Noto Sans JP','Hiragino Sans','Yu Gothic',sans-serif` |
- 임베드는 **OFL/CC0만**(Pretendard·Noto·Inter 가능). 유료 폰트 금지. `font-display:swap`+preload, 로케일별 서브셋/지연로딩으로 용량 관리.
- 줄높이: CJK(ko/zh/ja)는 1.5~1.6, 라틴(en)은 1.45~1.55 권장(가독). 토큰에서 로케일별 조정.

## 5. 텍스트 길이 변동(Text Expansion) — 레이아웃 규칙
번역은 길이가 다르다(영어가 길어지는 경우 多, 일부 CJK는 짧음). 시네마틱 레이아웃이 깨지지 않도록:
- **고정 높이/폭 금지.** 컨테이너는 내용에 따라 늘어나고, `clamp()` 타이포·`min-height` 사용.
- 버튼/칩/KPI 라벨은 **줄바꿈 허용**·말줄임(`text-overflow`) 대비. 1줄 가정 금지.
- 큰 타이틀은 언어별로 줄수 달라질 수 있음 → 컨테이너가 수용. 모션은 줄수에 비종속.
- 타이밍: 텍스트 등장 애니메이션은 길이에 비례하지 않게(또는 최대치 캡). 자동진행 없음(사용자 클릭) 권장.
- QA는 4개 언어 × (320/390/768/1280) 매트릭스로.

## 6. 번역 톤 가이드 (언어별)
정서가 핵심인 작품 → **직역 금지, 임팩트 우선.** 명제·결말 라인은 특히 자연스럽고 묵직하게.
- `ko`: 정본. 담담하고 단정, 마지막 문장은 여운.
- `en`: 간결·문학적. 수동/단문으로 무게. 의료·정책 용어는 표준 영어(e.g., "deskilling", "verification capacity").
- `zh`: 简体, 书面语. 정확하고 절제된 톤. 감정선은 유지하되 과장 금지.
- `ja`: です/ます체 기반의 담담한 서술 + 결정적 문장은 짧게. 한자/가나 균형.
- 숫자·고유표기: AF447, Lancet, FAA, MIT, Stanford 등 **원표기 유지**. 연도·퍼센트는 로케일 숫자 포맷(대개 동일).
- 고지문("등장인물·기관은 가상")은 각 언어 표준 면책 톤으로.

## 7. 언어 전환 UX
- 위치: 상단 바 우측(음소거 토글 옆) 또는 인트로(S0) 하단. 작은 드롭다운/세그먼트.
- 라벨은 **각 언어의 자기표기**: `한국어 / English / 中文 / 日本語`.
- 전환 즉시 전체 텍스트·`<html lang>`·폰트 교체. 진행 상태 유지.
- 접근성: `aria-label`, 키보드 접근, 현재 선택 표시.

## 8. 메타/SEO 다국어
- `<html lang="{locale}">` 동적.
- `hreflang` 대체 링크(ko/en/zh/ja) + `og:locale` / `og:locale:alternate`.
- OG 미리보기 이미지: 언어별 1장 권장(최소 en/ko). 텍스트는 자작.

## 9. QA 매트릭스 (현지화)
- [ ] 4개 언어 × 4개 폭에서 깨짐 0, 가로 스크롤 0, 잘림 0.
- [ ] 모든 키가 4개 파일에 존재(누락 0). 미번역 키 0.
- [ ] 강조 토큰(`<em>/<gn>/<rd>`)이 각 언어에서 의미 위치에 유지.
- [ ] 폰트가 로케일별로 정확히 적용(자형 확인), FOUT 최소.
- [ ] 전환이 진행 중에도 매끄럽고 상태 유지.
- [ ] reduced-motion에서 4개 언어 모두 완주.
