# CLAUDE.md — 에이전트 운영 지침

## 프로젝트
「마지막 검증자」 — 2026 KAIST AI × 실패 아이디어 공모전 **보완 자료**. 선택지로 진행되지만 각 장면이 **모션그래픽 영상처럼** 재생되는 모바일·데스크톱 반응형 웹 경험(약 3–5분). 정적 빌드 → 외부 링크 제출.

## 무엇을 먼저 읽나 (우선순위)
1. `PRD.md` — 무엇을/왜.
2. `STORYBOARD.md` — **콘텐츠 정본**(장면·카피·연출·분기).
3. `TASKS.md` — Phase 0부터 순서대로 구현.
4. 막히면: 콘텐츠=STORYBOARD, 디자인=DESIGN_SYSTEM, 기술=TECH_SPEC.

## 절대 규칙 (위반 금지)
1. **저작권/IP:** 외부 영상·사진·음악·유료 폰트·캐릭터·로고·실존 인물 자산 **금지**. 허용: 자작 SVG/벡터, CSS/Canvas 생성, **CC0/OFL** 사운드·폰트. 모든 에셋 출처를 `public/CREDITS.md`에 기록. (공모전 '순수 창작물·타인 저작권/초상권 비침해' 요건.)
2. **카피 고정:** `STORYBOARD.md`의 한국어 텍스트를 **임의로 바꾸지 않는다**. 오타/개선이 보이면 코드 코멘트(`// COPY-SUGGESTION:`)로 제안만.
3. **인물 추상화:** 김재민·박지훈 등은 이름·실루엣·상징만. **사실적 얼굴/사진 금지**. 실명 병원/실존 인물 금지. "등장인물·기관은 가상" 고지 유지.
4. **반응형·접근성:** 모바일 세로 우선, 데스크톱 대응. `prefers-reduced-motion` 지원 필수. 빠른 점멸/스트로브 금지(광과민). 오디오는 **첫 사용자 제스처에 활성 + 음소거 토글** + 자막/시각 대체. (기본 재생/음소거 여부는 제품 결정 사항 — 공모전 필수 요건 아님. **현재 방침 = 기본 재생**.)
5. **정적·무서버:** 계정/DB/백엔드 금지. 100% 정적 프런트엔드.
6. **민감성:** 환자 장애·고통은 절제해 표현(자극·과장 금지).

## 기술 컨벤션
- Vite + TypeScript(strict). 애니메이션은 GSAP(무료 범위) 또는 동등 OSS.
- 씬은 **데이터 주도**: 카피/분기는 `src/data/script.ts`(STORYBOARD 미러), 씬 모듈은 렌더+애니메이트.
- 모션은 **transform/opacity 위주**(레이아웃 thrash 회피). reduced-motion 분기는 `motion.ts`에 일원화.
- 컴포넌트/씬 1파일 단위, 명확한 `enter()/exit()`로 타임라인·리스너 정리(리플레이 누수 0).
- 상태는 `engine/state.ts` 단일 출처. 결말 분기: `vCap<35 비극 / 35~69 near-miss / ≥70 회피`.

## 실행/빌드
```
npm i
npm run dev      # 개발
npm run build    # dist/ 생성
npm run preview  # 빌드 미리보기
```
배포: GitHub Pages 또는 Vercel(정적). Vite `base` 경로 주의. 링크는 '모든 사용자 열람 가능'.

## 작업 방식
- **TASKS.md의 Phase 순서대로.** 각 Phase의 DoD를 통과한 뒤 다음으로.
- 각 Phase 종료 시 모바일(360px)·데스크톱(1280px)에서 동작 확인.
- 큰 결정/모호함은 추정해 진행하되, 결정 사항을 커밋 메시지/주석에 남긴다.
