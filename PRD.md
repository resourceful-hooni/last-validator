# PRD — 「마지막 검증자」 인터랙티브 시네마틱 경험

문서 버전 1.0 · 대상: Claude Code 구현 에이전트 + 검수자

---

## 1. 개요 / 비전
2036년 한 응급실에서 벌어진 의료 실패를 **거꾸로 추적(Pre-Mortem)**하는 인터랙티브 경험. 플레이어는 2026년의 의사결정권자가 되어 다섯 번의 *합리적인* 선택을 내리고, 그 선택이 어떻게 2036년의 비극으로 이어지는지를 **영화처럼 연출된 장면**으로 마주한다.

핵심 정서: *"나는 모든 걸 옳게 했다. 모든 지표가 초록불이었다. 그래서 망했다."*

## 2. 배경 / 문제
- 제안서 핵심 명제: **"AI는 일자리를 빼앗지 않았다. 그것을 검증할 능력을 비웠다. 그래서 AI가 틀린 순간, 그 오류를 잡아낼 인간이 남지 않았다."**
- 가장 날카로운 통찰: **이 실패는 자기 경보를 스스로 끈다.** AI 의존이 커질수록 보조 지표(일치율·처리량)는 초록불이 되고, 붕괴를 감지할 유일한 계기(독립적 인간 판단)는 사라진다.
- 왜 '영상형 인터랙티브'인가: 1페이지 제안서가 못 주는 **체험과 정서**를 전달해 심사 기준의 '전달력'과 '통찰력'을 증폭. 정적 텍스트로는 "초록불의 함정"을 *느끼게* 할 수 없다.

## 3. 목표 / 비목표
**목표 (Goals)**
- G1. 핵심 통찰("초록불일 때 붕괴가 시작된다")을 **선택 → 즉각적 시각 보상 → 뒤늦은 대가**의 루프로 *체감*시킨다.
- G2. 영화 같은 모션(시네마틱 전환·타이포 애니메이션·계기판 글로우·CT 스캔 연출)으로 "영상이 흐르는" 느낌을 준다.
- G3. 모든 분기 결말이 결국 **같은 교훈**으로 수렴한다(리플레이로 함정을 재확인).
- G4. 실제 신호(2024 전공의·2025 Lancet·2009 AF447)로 **허구가 아님**을 못 박는다.
- G5. 모바일·데스크톱에서 매끈하게 동작, 링크 하나로 공유.

**비목표 (Non-Goals)**
- NG1. 실사 영상 촬영·실제 배우·실존 인물/기관 사용 (전부 추상·벡터·가상).
- NG2. 복잡한 게임 시스템(점수 경쟁, 멀티 엔딩 12종 등). 짧고 강렬(5~7장)이 원칙.
- NG3. 계정·서버·DB. 100% 정적 프런트엔드.
- NG4. 사실적 의료 시뮬레이션. 임상 디테일은 *상징적*으로만.

## 4. 타깃 & 사용 맥락
- 서비스 언어: **한국어·English·中文(简体)·日本語 4개 (P0)** — 로케일 전환 지원.
- 1차 사용자: **공모전 심사위원**(링크로 열람) 및 4개 언어권 일반 대중.
- 2차: 일반 대중(캠페인 확장 가능성).
- 기기: 모바일(세로) 우선, 데스크톱 대응. 터치/클릭.
- 소요: **3~5분**, 한 번에 끝까지. 이탈 방지가 핵심(진행바·짧은 장면).
- 환경: 외부 호스팅 링크, 오프라인에서도 동작(정적), 저사양 폰 고려.

## 5. 핵심 콘셉트 & 메커니즘
**2막 구조**
- ACT 1 (2026 · 결정권자): 5개의 의사결정. 각 결정은 두 갈래 — *효율 우선* vs *역량 보존*.
- ACT 2 (2036 · 목격자): 누적된 '독립 검증 역량(vCap)' 값에 따라 결말 분기.

**초록불의 함정 (핵심 장치)**
- 화면 상단에 **운영 대시보드**(4개 KPI: 판독 처리량·비용 절감·AI 일치율·환자 대기)가 상시 노출.
- *효율 우선* 선택 → 관련 KPI가 **초록불**로 점등(글로우 애니메이션) + vCap −20%.
- *역량 보존* 선택 → KPI는 호박색(효율 손해) + vCap 유지.
- 화면 구석 '독립 검증 역량' 바는 **작고 눈에 안 띄게** 표시되다가, **결정 4에서 효율을 택하면 회색 '측정 안 함'으로 꺼진다** = 경보가 스스로 꺼지는 연출.

**대가 (vCap → 결말)**
- vCap < 35: 비극(환자 영구장애). 35~69: 운에 맡긴 near-miss. ≥70: 회피(단, 대시보드는 끝내 완벽한 초록은 아니었음 — 트레이드오프 명시).

## 6. 기능 요구사항 (FR) — 우선순위 P0(필수)/P1(중요)/P2(여유)
- **FR-1 (P0) 오프닝/타이틀**: 시네마틱 타이틀 등장, 컨셉 1문장, [시작].
- **FR-2 (P0) 운영 대시보드**: 4 KPI + vCap 바. 상태(중립/초록/호박/꺼짐) + 점등/감소 애니메이션.
- **FR-3 (P0) 씬 플레이어**: 각 장면이 *애니메이션 시퀀스*로 재생(인트로 모션 → 정보 → 선택). 자동 스크롤/포커스.
- **FR-4 (P0) 선택 시스템**: 2지선다, 선택 후 즉시 결과 모션(KPI 변화 + 짧은 내레이션 + 델타 칩).
- **FR-5 (P0) 분기/상태**: vCap·greens·choices 누적. 결말 분기 로직.
- **FR-6 (P0) ACT1 결산**: "성공적이었습니다"(초록 스코어보드) → "아무도 이 숫자는 보지 않았습니다" → vCap 드라마틱 공개.
- **FR-7 (P0) ACT2 시네마틱**: 2036 시간점프 → 응급실 장면(분기 3종). CT 스캔/AI 오버레이/골든타임 시계 연출.
- **FR-8 (P0) 엔딩**: 역추적(선택→결과 애니메이션) + 실제 신호 앵커 + 명제 + CTA + 리플레이.
- **FR-9 (P1) 사운드**: 앰비언트 험, UI 틱, '초록불' 차임, 2036 긴장 드론 (전부 CC0/생성, 음소거 토글, 기본 OFF 권장).
- **FR-10 (P1) 진행 표시 & 스킵**: 장면 진행바, 모션 스킵/즉시진행, 리플레이.
- **FR-11 (P1) 공유**: 결과 화면에 공유 링크/제안서 링크.
- **FR-12 (P2) MP4 트레일러**: Remotion으로 60~90초 영상 렌더(별도 타깃, TECH_SPEC 참조).
- **FR-13 (P0) 다국어 i18n**: 한국어·영어·중국어(간체)·일본어 4개 언어. 언어 전환 UI, 로케일 자동감지+지속, 키 기반 텍스트(하드코딩 금지), 언어별 폰트. 상세는 `I18N.md` / 번역은 `TRANSLATIONS.md`.

콘텐츠(장면별 정확한 카피·연출)는 전부 `STORYBOARD.md` 참조 — 본 PRD는 구조만 규정.

## 7. 비기능 요구사항 (NFR)
- **반응형**: 320px~데스크톱. 모바일 세로 우선. 터치 타깃 ≥44px.
- **성능**: 모바일에서 60fps 목표, 초기 로드 < 3초(3G 기준 노력), 총 에셋 가볍게(가능하면 < 5MB).
- **접근성**: `prefers-reduced-motion` 시 모션 대폭 축소/즉시 표시. 텍스트 대체. 충분한 명도대비. 오디오엔 자막/설명.
- **호환**: 최신 모바일/데스크톱 브라우저(크롬·사파리·엣지). iOS 사파리 100dvh 처리.
- **오프라인/정적**: 서버 불필요. 단일 정적 빌드.
- **견고성**: 빠른 연타·중간 이탈/리로드에도 깨지지 않음. 리플레이는 상태 완전 초기화.

## 8. 성공 기준
**심사 기준 매핑**
- 통찰력: 초록불 함정 메커니즘 + "경보가 꺼진다" 연출이 또렷이 전달되는가.
- 인과성: 엔딩 역추적이 각 선택→결과를 명확히 잇고, 실제 신호로 뒷받침되는가.
- 현재가치: 제안서의 처방으로 연결되는 CTA가 있는가.
- 전달력: 끝까지 보게 되는가(이탈 적음), 정서적 잔상이 남는가.

**UX 지표(자체 검수)**
- 3~5분 내 완주 가능. 모바일/데스크톱 양쪽 깨짐 0.
- reduced-motion에서도 서사 완결.
- 비전문가도 핵심 메시지를 30초 안에 요약 가능.

## 9. 제약 (Constraints) — 매우 중요
- **저작권/IP**: 외부 영상·사진·음악·유료 폰트·캐릭터·로고·실존 인물 사진 **사용 금지**. 허용: 자작 SVG/벡터, CSS/Canvas 생성 그래픽, **CC0** 사운드/폰트(예: 시스템 폰트, Pretendard OFL 등 라이선스 확인). 공모전 '순수 창작·타인 저작권/초상권 비침해' 요건과 직결.
- **카피 고정**: STORYBOARD.md의 한국어 텍스트는 임의 변경 금지(오타/개선은 코드 코멘트로 제안).
- **사실성 경계**: 임상 디테일은 상징적으로만. 실명 병원/실존 인물 금지. "등장인물·기관은 가상" 고지 포함.
- **민감성**: 환자 장애·고통 묘사는 절제(자극·과장 금지). 음향·점멸은 광과민 위험 회피(빠른 깜빡임 금지).

## 10. 산출물 & 배포
- 정적 웹 빌드(`dist/`) → GitHub Pages(레포 `index.html`) 또는 Vercel.
- README에 빌드/배포 절차. 배포 링크는 **'모든 사용자 열람 가능'**.
- (옵션) `trailer.mp4` (Remotion render).
- 첫 화면 또는 호스팅 페이지 상단에 "본 자료는 1페이지 제안서의 보완 자료입니다" 표기.

## 12. 다국어(i18n) 요구 (P0 · 요약)
- 지원: **ko·en·zh·ja** 4개. 정본 ko, 폴백 ko.
- 모든 사용자 텍스트는 **키 기반**(하드코딩 0). 번역 전문은 `TRANSLATIONS.md` → `src/locales/*.json`.
- 언어 전환 UI(자기표기 라벨), 자동감지(navigator.language)+`localStorage` 지속, **진행 중 전환 허용**.
- 언어별 폰트(ko=Pretendard/Noto KR, en=Inter, zh=Noto SC, ja=Noto JP) + `<html lang>` 동적.
- 텍스트 길이 변동에 견디는 레이아웃(고정 폭/높이 금지). 메타: hreflang/og:locale.
- 성공기준에 **4개 언어 × 4개 폭 QA 통과** 추가. 상세 규칙은 `I18N.md`.
